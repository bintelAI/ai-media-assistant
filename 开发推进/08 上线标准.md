# 上线达成目标：
1. 产品功能正常，无重大问题
2. 可以正常登入
3. 产品至少看上去是完全可用



# 功能优化：
1. 禁用页面不可用的功能，可以在设置里面添加一个 dev开关功能， 开启后页面不可用的功能会被启用，默认情况下没有开发好的功能直接禁用，可以显示，但是采用灰色模式禁用
2. 使用 UI 技能，对页面进行整体优化 
3. 支持的平台，如果不支持的禁用灰色一下 
4. 在设置里面配置一个接口采集的间隔，默认随机 3-6秒一次接口，其他所有对接接口的采集全部使用这个设置，避免对服务器造成过大压力
5. 在批量采集那里，也是需要使用设置的随机间隔，且导入文件要说明清楚是什么类型，如果没必要，可以禁用导入 
6. 用户在采集的时候要有一个进度条，可以进行展示，否则客户不确认是否在采集。


7. 在维表里面创建一个项目，为我们的产品使用，就是 智联分析，产品采用 c 端模式 
    - 建立标准 C 端库，字段类型必须正确设置（数值字段用 number、时间字段用 date、链接字段用 url 等），确保后续可以正常筛选和排序
    - 公告在里面里面建一个表 ，对接页面的公告
    - 走马灯，对接图片，也在系统里面建一个表 ，插件可以点击跳转页面
    - 在数据入库完成后，"完成"按钮变为跳转链接，点击直接打开维表中的目标表 

# bug列表：
1. 维表自动创建的表（帖子、作者、评论），所有字段类型全部设置为 text，包括点赞数、评论数、粉丝数等数值字段，导致用户无法在维表中进行数值筛选、排序等操作
2. 维表入库完成后没有跳转到目标表的链接，用户需要手动在维表中找到对应表，体验不完整




在数据入库 分为 2 大块：
    1. 入C 端系统库 
        写死存储数据 ，这个库的字段类型必须正确设置（数值字段用 number、时间字段用 date、链接字段用 url 等），确保后续可以正常筛选和排序
    2. 入自有的数据库
        弹窗设置，双选
