## 四、落地实施建议（按优先级排列）

### P0 — 阻塞问题（必须先解决，否则无法上线）

#### 1. 建立标准 C 端维表库，修复字段类型

**涉及文件**：
- `src/shared/services/dimens-service.ts` — `SHEET_CONFIGS` 字段类型定义
- `src/shared/services/dimens-service.ts` — `createColumn` 和 `ensureColumns` 函数

**根因**：当前 `SHEET_CONFIGS` 中所有字段的 `type` 均为 `'text'`，包括 `likeCount`、`fansCount` 等数值字段和 `publishTime`、`collectedAt` 等时间字段，导致维表无法按数值筛选和排序。

**方案**：
1. 重新定义 `SHEET_CONFIGS` 中字段的维表类型，建立标准 C 端库字段规范：

   | 维表字段类型 | 适用字段 | 说明 |
   |-------------|---------|------|
   | `text` | 标题、内容描述、标签、备注等 | 文本类字段 |
   | `number` | likeCount, commentCount, fansCount, viewCount 等所有数值 | 支持数值筛选、排序、聚合 |
   | `date` | publishTime, collectedAt, updatedAt 等所有时间字段 | 支持按时间范围筛选、排序 |
   | `url` | url, coverUrl, profileUrl, avatar 等链接字段 | 支持点击跳转 |
   | `select` | platform, postType, gender, verified 等枚举字段 | 支持下拉选择筛选项 |

2. 修改 `SHEET_CONFIGS` 中每列的 `type` 为正确的维表类型
3. 更新 `createColumn` 调用，确保维表 API 传递正确的字段类型
4. 更新 `createStandardSheetForType` 方法，创建标准表时字段类型自动匹配

#### 2. 添加 Dev 开关 + 功能禁用系统

**涉及文件**：
- `src/shared/utils/constants.ts` — 定义功能开关列表及状态
- `src/shared/store/settingsStore.ts` — 添加 `devMode` 和 `featureFlags` 状态
- `src/entrypoints/sidepanel/pages/Settings.tsx` — 添加 Dev 开关界面
- `src/entrypoints/sidepanel/components/Navigation.tsx` — 根据开关控制导航项显示
- `src/entrypoints/sidepanel/pages/Overview.tsx` — 平台卡片根据开关显示禁用态

**方案**：
1. 在 `constants.ts` 中定义 `FEATURE_FLAGS` 常量，标注每个功能的状态（`ready` / `disabled` / `beta`）
2. 在 `settingsStore` 中添加 `devMode: boolean` 开关，默认 `false`
3. 默认情况下，状态为 `disabled` 的功能在界面上灰显（`opacity-50` + `cursor-not-allowed`），不可点击
4. 开启 Dev 模式后，所有功能恢复可用
5. 当前可标记为 `disabled` 的功能：抖音采集、快手采集、TikTok采集、星图采集、批量导入

## 六、风险提示

| 风险 | 说明 | 缓解措施 |
|------|------|---------|
| 字段类型兼容性 | 维表 API 的字段类型可能随版本变化，`number`/`date`/`url`/`select` 类型在各版本间可能表现不一致 | 在上线前用测试账号创建标准表验证所有字段类型的筛选和排序功能正常 |
| 采集频率过高 | 当前批量采集间隔硬编码，无用户控制，可能触发平台风控 | 快速实现采集间隔设置并默认 3-6秒 |
| 未完成平台暴露 | 抖音、快手等 content script 未完全实现，用户点击可能报错 | 通过 Dev 开关默认禁用，上线时只暴露已验证的平台 |
| 数据存储限制 | IndexedDB 存储空间受浏览器限制，大量数据可能导致写入失败 | 在设置中展示存储用量，提供数据导出后清空的建议提示 |
