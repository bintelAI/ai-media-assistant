## 二、分层职责

### 1）Content Script 层

职责：

- 识别当前页面是什么类型
  - 帖子页
  - 作者页
  - 列表页
  - 评论页
- 注入操作按钮
- 读取页面 DOM / 页面状态
- 把原始数据发送给后台或侧栏

不建议在这里做太多：
- 不做复杂状态管理
- 不做大批量数据存储
- 不做复杂导出

### 2）Background 层

职责：

- 管理任务队列
- 统一接收采集请求
- 调度下载任务
- 协调本地存储
- 广播任务状态给 Side Panel

这是插件的"调度中枢"。

### 3）Side Panel 层

职责：

- 显示采集结果
- 数据筛选和管理
- 导出配置 UI
- 下载和任务中心 UI
- 设置页面

这是"工作台"。

### 4）Platform Adapter 层

职责：

- 针对不同平台实现统一接口

建议统一接口：

- `matchPage()`
- `getPageType()`
- `extractPost()`
- `extractAuthor()`
- `extractComments()`
- `extractPostList()`
- `extractMedia()`

这样可以避免平台逻辑散落 everywhere。

### 5）Storage 层

建议分两层：

#### `chrome.storage.local`

适合放：
- 设置项
- 轻量状态
- 最近使用模板
- 当前任务摘要

#### `IndexedDB`

适合放：
- 帖子/作者/评论大数据
- 导出数据集
- 任务历史
- 下载记录

原因：
- 数据量更大
- 查询更适合列表场景
- 比 storage 更适合本地数据库角色

## 五、IndexedDB 表设计

### 1. 数据库初始化

使用 `idb` 库进行 IndexedDB 操作。

### 2. 表结构设计

- Posts 表：包含平台、帖子ID、标题、内容等字段
- Authors 表：包含平台、作者ID、昵称、粉丝数等字段
- Comments 表：包含平台、评论ID、内容、所属帖子等字段
- Media 表：包含媒体类型、URL、下载状态等字段
- Tasks 表：包含任务类型、状态、进度等字段
- Templates 表：包含导出模板配置

### 3. 索引设计

每个表需要设计合适的索引以支持查询：

- 主键索引：id
- 平台索引：platform
- 时间索引：collectedAt
- 唯一性索引：platform + postId / authorId / commentId

## 七、平台适配器接口定义

### 1. 适配器类型定义

```typescript
// src/shared/adapters/types.ts

export type Platform = 'xhs' | 'dy' | 'ks' | 'tiktok'

export type PageType = 
  | 'post_detail'     // 帖子详情页
  | 'author_profile'  // 作者主页
  | 'comments_page'   // 评论页
  | 'feed_list'       // 列表流页
  | 'search_result'   // 搜索结果页
  | 'unknown'         // 未识别

export interface PlatformAdapter {
  // 基本信息
  name: Platform
  displayName: string
  
  // 页面识别
  detectPage(url: string): PageType
  
  // 数据抽取
  extractPost(): Promise<ExtractResult<PostEntity>>
  extractAuthor(): Promise<ExtractResult<AuthorEntity>>
  extractComments(): Promise<ExtractResult<CommentEntity[]>>
  extractPostList(): Promise<ExtractResult<PostEntity[]>>
  extractMedia(): Promise<ExtractResult<MediaEntity[]>>
}
```

### 2. 适配器统一接口

每个平台适配器需要实现：

- `detectPage(url)`：识别页面类型
- `extractPost()`：抽取帖子数据
- `extractAuthor()`：抽取作者数据
- `extractComments()`：抽取评论数据
- `extractPostList()`：抽取帖子列表
- `extractMedia()`：抽取媒体资源

## 十一、关键技术点说明

### 1. SPA 页面监听

小红书、抖音等平台是 SPA，需要监听路由变化：

- 使用 MutationObserver 监听 DOM 变化
- 监听 popstate 事件
- 定时检查 URL 变化

### 2. 数据抽取策略

优先级：

1. 从页面状态对象（如 `__INITIAL_STATE__`）抽取
2. 从 DOM 元素抽取
3. 从网络请求拦截抽取

### 3. 去重机制

基于 `platform + postId` 唯一键去重：

- 新数据入库前先查询是否存在
- 存在则更新，不存在则新增

### 4. 导出文件生成

使用 SheetJS：

- Excel：`XLSX.utils.json_to_sheet()` + `XLSX.writeFile()`
- CSV：`XLSX.utils.sheet_to_csv()` + Blob 下载
- JSON：`JSON.stringify()` + Blob 下载

### 5. 虚拟滚动

使用 `@tanstack/react-virtual` 处理大数据列表。

## 十二、开发约束

### 1. 类型安全

所有数据结构必须有 TypeScript 类型定义。

### 2. 错误处理

所有异步操作必须有错误捕获和用户提示。

### 3. 性能优化

- 长列表使用虚拟滚动
- 大数据导出分批处理
- 避免在 Content Script 中存储大量数据

### 4. 样式隔离

注入到页面的 UI 使用 Shadow DOM 防止样式冲突。

## 十三、扩展性设计

### 1. 平台扩展

通过 Platform Adapter 接口，新增平台只需：

- 实现适配器接口
- 注册到适配器注册表
- 配置页面识别规则

### 2. 字段扩展

通过 ExportFieldConfig 配置，新增字段只需：

- 在数据字典中定义
- 在导出字段配置表中添加
- 在 UI 中自动渲染

### 3. 功能扩展

通过消息类型和任务类型扩展，新增功能只需：

- 定义新的消息类型
- 实现对应的 Handler
- 在 UI 中添加入口
