## 模块 A：页面采集模块

这是插件的核心入口模块。

### A1. 采集对象类型

支持以下采集类型：

1. **帖子采集**
   - 帖子标题
   - 帖子正文/描述
   - 发布时间
   - 帖子链接
   - 帖子 ID
   - 点赞/评论/收藏/分享等可见指标
   - 媒体资源信息
   - 所属作者基础信息

2. **作者采集**
   - 作者 ID
   - 昵称
   - 主页链接
   - 粉丝数
   - 获赞数/作品数等可见指标
   - 简介
   - 头像

3. **评论采集**
   - 评论 ID
   - 评论内容
   - 评论用户
   - 评论时间
   - 点赞数
   - 回复数
   - 所属帖子 ID

4. **作者作品列表采集**
   - 作者主页下的作品列表
   - 每条作品的标题/封面/链接/发布时间/互动指标

### A2. 采集触发方式

支持三种触发入口：

1. 页面注入按钮
   - 帖子页："采集帖子"
   - 作者页："采集作者"
   - 评论区："采集评论"
   - 列表页："批量采集当前列表"

2. 右键菜单（可选，P2）
   - 对链接/页面执行采集

3. Side Panel 中手动执行
   - 读取当前页面并识别可采集对象

### A3. 采集结果要求

- 采集完成后写入本地数据库
- 页面给出成功/失败反馈
- 支持重复采集检测
- 支持增量采集标识

---

## 模块 B：任务中心模块

用于处理批量采集和下载。

### B1. 任务类型

- 单条帖子采集
- 单个作者采集
- 评论采集
- 作者作品列表采集
- 媒体下载任务
- 导出任务

### B2. 任务状态

统一状态：

- 待执行
- 执行中
- 已成功
- 已失败
- 已取消

### B3. 任务能力

- 添加任务
- 批量添加任务
- 顺序执行
- 并发数配置
- 失败重试
- 暂停/继续
- 清空已完成任务
- 查看失败原因

### B4. 当前阶段建议

前期先做简单版本：

- 单线程或小并发
- 本地任务队列
- 基础重试 1~2 次
- 不做复杂调度

---

## 模块 C：本地数据管理模块

这是导出能力的前置基础。

### C1. 数据分类

在本地分成几类数据表：

- 帖子数据
- 作者数据
- 评论数据
- 媒体数据
- 任务记录

### C2. 数据能力

- 列表查看
- 搜索
- 筛选
- 排序
- 去重
- 批量删除
- 批量选择
- 查看详情

### C3. 去重规则

建议先定义：

- 帖子：`platform + postId`
- 作者：`platform + authorId`
- 评论：`platform + commentId`
- 媒体：`platform + mediaUrl 或 mediaId`

### C4. 标签与备注

当前阶段建议做轻量能力：

- 收藏/标星
- 自定义备注
- 来源页面记录
- 采集时间记录

---

## 模块 D：导出模块

这是当前优先解决的模块。

### D1. 支持导出格式

优先级建议：

1. CSV
2. Excel xlsx
3. JSON

### D2. 导出对象

- 导出帖子列表
- 导出作者列表
- 导出评论列表
- 导出当前筛选结果
- 导出当前选中结果
- 导出单条详情

### D3. 导出能力

- 字段勾选
- 字段顺序调整
- 字段中文名自定义
- 空值处理
- 日期格式化
- 文件名规则配置

### D4. 导出模板

建议前端支持模板：

- 帖子导出模板
- 作者导出模板
- 评论导出模板

每个模板包含：
- 选中的字段
- 字段顺序
- 字段显示名
- 文件格式

### D5. 导出限制

当前阶段建议：
- 只做本地文件导出
- 不做云端上传
- 不做多维表同步
- 不做自动分享

---

## 模块 E：媒体下载模块

### E1. 能力范围

- 下载单条帖子媒体
- 批量下载选中帖子媒体
- 支持图片/视频
- 文件命名规则
- 下载状态展示

### E2. 本期建议

先做：

- 单视频下载
- 多图下载
- 批量加入下载任务
- 使用 Chrome downloads API

暂不做：
- 超复杂目录规则
- 断点续传
- 超大批量 ZIP 打包

- 任务并发数

- 启用/禁用平台支持
- 某平台采集按钮显示开关
- 某平台实验功能开关

### F3. 数据设置

- 数据清理
- 清空任务记录
- 清空本地缓存
- 导出本地数据库备份（P2）

## 导出模块详细说明

### 1. 导出模块目标

让用户能从本地数据中快速生成结构化文件，并满足不同使用习惯：

- 直接导出给 Excel 用户
- 导出 JSON 给技术同学
- 按筛选结果导出给运营团队
- 批量导出选中的内容与字段

### 2. 导出入口

导出入口需要统一，但不要太分散。

#### 入口 1：列表页顶部

- 导出全部
- 导出筛选结果
- 导出选中项

#### 入口 2：详情页

- 导出单条

#### 入口 3：任务中心

- 某任务完成后快捷导出（P1）

### 3. 导出流程

统一导出流程如下：

```text
点击导出
  ↓
打开导出弹窗
  ↓
选择导出范围
  ↓
选择文件格式
  ↓
选择字段
  ↓
配置文件名与模板
  ↓
确认导出
  ↓
后台生成文件
  ↓
浏览器下载
```

### 4. 导出弹窗详细结构

#### 4.1 弹窗标题区

- 标题：导出数据
- 副标题：当前导出对象类型 + 数量

例如：
- 导出帖子数据（共 128 条）
- 导出作者数据（已选中 12 条）

#### 4.2 区块一：导出范围

单选项：

- 导出全部数据
- 导出当前筛选结果
- 导出当前选中数据

#### 规则

- 如果没有筛选条件，则"当前筛选结果"可隐藏或禁用
- 如果没有选中数据，则"选中数据"禁用

#### 4.3 区块二：导出格式

单选项：

- CSV
- Excel (.xlsx)
- JSON

#### 默认值

读取用户设置中的默认导出格式。

#### 4.4 区块三：字段选择

按分组展示字段，支持全选/取消全选。

#### 帖子导出字段示例

**基础字段**
- 标题
- 内容
- 帖子链接
- 帖子 ID
- 平台
- 发布时间

**作者字段**
- 作者 ID
- 作者昵称
- 作者主页链接

**互动字段**
- 点赞数
- 评论数
- 收藏数
- 分享数

**媒体字段**
- 媒体类型
- 封面图
- 媒体链接列表

**系统字段**
- 来源页面
- 采集时间
- 更新时间

#### 字段交互

- 勾选启用字段
- 支持拖拽排序
- 支持修改列名

#### 4.5 区块四：字段映射与列名

每个字段旁边增加：

- 原字段名
- 导出列名输入框

例如：

| 原字段 | 导出列名 |
|---|---|
| title | 标题 |
| authorName | 作者昵称 |
| likeCount | 点赞数 |

这样后期能支持不同团队导出习惯。

#### 4.6 区块五：文件设置

#### 配置项

- 文件名输入框
- 自动附加日期后缀 开关
- 日期格式选择
- 编码设置（CSV 可选 UTF-8 with BOM）

#### 默认文件名建议

- `posts_20260327`
- `authors_20260327`
- `comments_20260327`

#### 4.7 区块六：模板

#### 能力

- 保存当前配置为模板
- 选择已有模板
- 删除模板

#### 模板内容

- 导出对象类型
- 导出格式
- 字段勾选
- 字段顺序
- 自定义列名
- 文件名规则

#### 模板分类

- 帖子模板
- 作者模板
- 评论模板

#### 4.8 底部按钮区

- 取消
- 确认导出

导出中状态：
- 确认按钮变为 loading
- 禁止重复提交

### 5. 导出生成逻辑

#### 数据来源

从 IndexedDB 查询导出数据。

#### 数据处理步骤

1. 取出目标数据
2. 应用筛选/选中范围
3. 按字段配置映射
4. 格式化字段值
5. 生成目标文件
6. 通过浏览器下载

### 6. 导出错误处理

导出失败时应提示明确原因：

- 没有可导出的数据
- 所选字段为空
- 文件生成失败
- 数据格式异常
- 下载触发失败

错误提示建议直接面向用户：
- "当前没有可导出的数据"
- "请至少选择一个字段"
- "导出失败，请重试"

### 7. 导出模块验收标准

#### 功能验收

- 能导出帖子/作者/评论数据
- 能选择字段
- 能修改列名
- 能导出 CSV / Excel / JSON

#### 体验验收

- 导出流程不超过一个弹窗
- 用户不需要手工整理数据顺序
- 失败时有清晰提示
