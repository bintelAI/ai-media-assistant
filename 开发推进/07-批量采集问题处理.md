# 批量采集功能问题处理文档

## 一、问题背景

### 1.1 当前项目现状
当前项目（智联采集）已实现基础的单条数据采集功能：
- ✅ 支持小红书、抖音、快手等平台
- ✅ 通过XHR拦截自动缓存页面数据
- ✅ 在详情页和列表页可单条采集
- ✅ 数据存储到本地IndexedDB
- ✅ 侧边栏展示采集列表

### 1.2 存在的问题
**核心问题：缺少批量采集功能**

| 问题点 | 具体表现 | 影响 |
|--------|---------|------|
| 无批量采集入口 | 列表页只能逐个点击"采集"按钮 | 效率低下，用户体验差 |
| 无批量操作面板 | 无法选择多个项目一次性采集 | 无法满足批量处理需求 |
| 无采集进度反馈 | 批量采集时无进度提示 | 用户不知道采集状态 |
| 无批量选择功能 | 无法全选/反选/筛选 | 操作繁琐 |

## 二、需求分析

### 2.1 批量采集的两种实现方向

本项目需要支持两种批量采集方式：

| 方向 | 描述 | 适用场景 | 优势 |
|------|------|---------|------|
| **方向一** | 从当前页面缓存数据批量采集 | 用户在浏览列表页时 | 无需手动输入，快速采集当前看到的内容 |
| **方向二** | 通过批量输入URL地址采集 | 用户有明确的采集目标列表 | 精准采集，支持跨页面批量操作 |

---

### 2.2 方向一：页面缓存批量采集

#### 2.2.1 批量采集入口
**需求描述**：在列表页添加批量采集入口

**实现方式**：
- 方案A：浮动按钮（推荐）
  - 在页面右下角添加浮动按钮
  - 点击后弹出批量采集面板
  - 不影响页面原有布局
  
- 方案B：顶部工具栏
  - 在页面顶部注入工具栏
  - 包含批量采集按钮
  - 可能遮挡页面内容

**推荐方案**：方案A - 浮动按钮

#### 2.2.2 批量采集面板
**需求描述**：弹出面板控制批量采集

**面板功能**：
1. **数据列表展示**
   - 显示当前页面已缓存的所有帖子
   - 每条数据显示：封面、标题、作者、互动数据
   - 支持勾选/取消勾选

2. **批量操作**
   - 全选/反选
   - 按平台筛选
   - 按互动数据排序

3. **采集控制**
   - 开始采集按钮
   - 采集进度显示
   - 暂停/继续功能（可选）

4. **结果反馈**
   - 成功数量统计
   - 失败数量统计
   - 失败原因提示

#### 2.2.3 批量采集流程
```
用户点击浮动按钮
    ↓
弹出批量采集面板
    ↓
显示当前页面缓存的帖子列表
```

### 2.3 方向二：URL批量采集

#### 2.3.1 功能概述
**需求描述**：用户批量输入帖子或用户的URL地址，系统自动访问并采集数据

**核心特性**：
- ✅ 支持批量输入URL（文本框/文件导入）
- ✅ 自动识别平台和页面类型
- ✅ 随机间隔采集（3-10秒）避免被封
- ✅ 实时进度反馈
- ✅ 失败重试机制

#### 2.3.2 URL输入方式

**方式一：文本框输入**
```
https://www.xiaohongshu.com/explore/abc123
https://www.douyin.com/video/123456789
https://www.xiaohongshu.com/user/profile/xyz789
```

**方式二：文件导入**
- 支持 .txt 文件（每行一个URL）
- 支持 .csv 文件（包含URL列）
- 支持 .xlsx 文件（包含URL列）

#### 2.3.3 采集流程设计

```
用户输入URL列表
    ↓
系统解析URL
    ↓
识别平台和页面类型
    ↓
验证URL有效性
    ↓
创建采集任务队列
    ↓
逐个访问URL（随机间隔3-10秒）
    ↓
提取数据并保存
    ↓
更新进度和结果
```

#### 2.3.4 随机间隔机制

**实现原理**：
```typescript
// 随机间隔函数
function getRandomInterval(): number {
  const min = 3000;  // 3秒
  const max = 10000; // 10秒
  return Math.floor(Math.random() * (max - min + 1)) + min;
}

// 批量采集时使用
async function batchCollectWithInterval(urls: string[]) {
  for (let i = 0; i < urls.length; i++) {
    const url = urls[i];
    
    // 采集数据
    await collectFromUrl(url);
    
    // 更新进度
    updateProgress(i + 1, urls.length);
    
    // 随机间隔等待（最后一个不需要等待）
    if (i < urls.length - 1) {
      const interval = getRandomInterval();
      await sleep(interval);
    }
  }
}
```

**为什么需要随机间隔？**
- 避免频繁请求被平台识别为爬虫
- 降低账号被封禁的风险
- 模拟真实用户行为
- 减轻服务器压力

#### 2.3.5 URL解析规则

**小红书URL格式**：
```
帖子详情：https://www.xiaohongshu.com/explore/{noteId}
用户主页：https://www.xiaohongshu.com/user/profile/{userId}
```

**抖音URL格式**：
```
视频详情：https://www.douyin.com/video/{awemeId}
用户主页：https://www.douyin.com/user/{secUserId}
```

**快手URL格式**：
```
视频详情：https://www.kuaishou.com/short-video/{photoId}
用户主页：https://www.kuaishou.com/profile/{userId}
```

**URL解析代码示例**：
```typescript
interface ParsedUrl {
  platform: Platform;
  pageType: PageType;
  id: string;
  originalUrl: string;
}

function parseUrl(url: string): ParsedUrl | null {
  // 小红书
  const xhsPostMatch = url.match(/xiaohongshu\.com\/explore\/([\w]+)/);
  if (xhsPostMatch) {
    return {
      platform: 'xhs',
      pageType: 'post_detail',
      id: xhsPostMatch[1],
      originalUrl: url
    };
  }
  
  const xhsUserMatch = url.match(/xiaohongshu\.com\/user\/profile\/([\w]+)/);
  if (xhsUserMatch) {
    return {
      platform: 'xhs',
      pageType: 'author_profile',
      id: xhsUserMatch[1],
      originalUrl: url
    };
  }
  
  // 抖音
  const dyVideoMatch = url.match(/douyin\.com\/video\/(\d+)/);
  if (dyVideoMatch) {
    return {
      platform: 'douyin',
      pageType: 'post_detail',
      id: dyVideoMatch[1],
      originalUrl: url
    };
  }
  
  const dyUserMatch = url.match(/douyin\.com\/user\/(MS4wLjABAAAA[\w-]+)/);
  if (dyUserMatch) {
    return {
      platform: 'douyin',
      pageType: 'author_profile',
      id: dyUserMatch[1],
      originalUrl: url
    };
  }
  
  // 快手...
  
  return null;
}
```

#### 2.3.6 数据采集策略

**方案A：后台标签页采集（推荐）**
```typescript
async function collectFromUrl(url: string): Promise<CollectResult> {
  // 1. 在后台打开标签页
  const tab = await chrome.tabs.create({
    url: url,
    active: false  // 后台打开
  });
  
  // 2. 等待页面加载完成
  await waitForTabComplete(tab.id);
  
  // 3. 注入Content Script提取数据
  const results = await chrome.scripting.executeScript({
    target: { tabId: tab.id },
    func: extractDataFromPage,
    args: []
  });
  
  // 4. 关闭标签页
  await chrome.tabs.remove(tab.id);
  
  // 5. 返回采集结果
  return results[0].result;
}
```

**方案B：API请求采集**
```typescript
async function collectFromUrl(url: string): Promise<CollectResult> {
  const parsed = parseUrl(url);
  
  // 根据平台和类型调用对应API
  if (parsed.platform === 'xhs' && parsed.pageType === 'post_detail') {
    return await fetchXhsPostDetail(parsed.id);
  } else if (parsed.platform === 'xhs' && parsed.pageType === 'author_profile') {
    return await fetchXhsAuthorDetail(parsed.id);
  }
  // ...
}
```

**推荐方案**：方案A（后台标签页采集）
- 优势：无需逆向API，稳定性高
- 劣势：资源消耗稍大

#### 2.3.7 进度管理

**进度状态**：
```typescript
interface BatchCollectProgress {
  total: number;           // 总数
  current: number;         // 当前进度
  success: number;         // 成功数
  failed: number;          // 失败数
  currentUrl: string;      // 当前处理的URL

  results: CollectResult[]; // 采集结果列表

```

**进度更新机制**：
- 每完成一个URL采集，立即更新进度
- 通过消息通知UI层更新显示
- 支持暂停/继续操作

#### 2.3.8 错误处理

**错误类型**：
| 错误类型 | 原因 | 处理方式 |
|---------|------|---------|
| URL格式错误 | URL不符合规则 | 跳过并记录错误 |
| 页面加载超时 | 网络问题或页面响应慢 | 重试3次 |
| 数据提取失败 | 页面结构变化 | 记录错误并继续 |
| 账号被封 | 请求过于频繁 | 暂停采集并提示 |

**重试机制**：
```typescript
async function collectWithRetry(
  url: string,
  maxRetries: number = 3
): Promise<CollectResult> {
  let lastError: Error | null = null;
  
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await collectFromUrl(url);
    } catch (error) {
      lastError = error;
      console.warn(`采集失败，第${i + 1}次重试:`, url);
      await sleep(2000); // 重试前等待2秒
    }
  }
  
  return {
    success: false,
    url: url,
    error: lastError?.message || '未知错误'
  };
}
```

```typescript
export async function handleBatchCollect(
  message: { posts: PostEntity[] }
): Promise<{ success: boolean; results: CollectResult[] }> {
  const results: CollectResult[] = [];
  
  for (const post of message.posts) {
    try {
      await savePostToDB(post);
      results.push({ id: post.id, success: true });
    } catch (error) {
      results.push({ id: post.id, success: false, error: error.message });
    }
  }
  
  return { success: true, results };
}
```

```typescript
export function parseUrl(url: string): ParsedUrl | null {
  try {
    const urlObj = new URL(url.trim());
    
    // 小红书
    if (urlObj.hostname.includes('xiaohongshu.com')) {
      return parseXhsUrl(url);
    }
    
    // 抖音
    if (urlObj.hostname.includes('douyin.com')) {
      return parseDouyinUrl(url);
    }
    
    // 快手
    if (urlObj.hostname.includes('kuaishou.com')) {
      return parseKuaishouUrl(url);
    }
    
    return null;
  } catch (error) {
    return null;
  }
}

function parseXhsUrl(url: string): ParsedUrl | null {
  const postMatch = url.match(/xiaohongshu\.com\/explore\/([\w]+)/);
  if (postMatch) {
    return {
      platform: 'xhs',
      pageType: 'post_detail',
      id: postMatch[1],
      originalUrl: url
    };
  }
  
  const userMatch = url.match(/xiaohongshu\.com\/user\/profile\/([\w]+)/);
  if (userMatch) {
    return {
      platform: 'xhs',
      pageType: 'author_profile',
      id: userMatch[1],
      originalUrl: url
    };
  }
  
  return null;
}
```

```typescript
export class TabManager {
  private activeTabs: Map<number, chrome.tabs.Tab> = new Map();
  private maxConcurrent: number = 3; // 最大并发数
  
  async createBackgroundTab(url: string): Promise<number> {
    // 等待可用槽位
    while (this.activeTabs.size >= this.maxConcurrent) {
      await sleep(500);
    }
    
    const tab = await chrome.tabs.create({
      url: url,
      active: false
    });
    
    if (tab.id) {
      this.activeTabs.set(tab.id, tab);
    }
    
    return tab.id!;
  }
  
  async closeTab(tabId: number): Promise<void> {
    try {
      await chrome.tabs.remove(tabId);
      this.activeTabs.delete(tabId);
    } catch (error) {
      console.warn('关闭标签页失败:', error);
    }
  }
  
  async waitForTabComplete(tabId: number, timeout: number = 30000): Promise<void> {
    return new Promise((resolve, reject) => {
      const timer = setTimeout(() => {
        reject(new Error('页面加载超时'));
      }, timeout);
      
      const listener = (updatedTabId: number, changeInfo: chrome.tabs.TabChangeInfo) => {
        if (updatedTabId === tabId && changeInfo.status === 'complete') {
          clearTimeout(timer);
          chrome.tabs.onUpdated.removeListener(listener);
          resolve();
        }
      };
      
      chrome.tabs.onUpdated.addListener(listener);
    });
  }
  
  async extractDataFromTab(tabId: number): Promise<any> {
    const results = await chrome.scripting.executeScript({
      target: { tabId: tabId },
      func: () => {
        // 在页面上下文中执行的数据提取逻辑
        // 返回提取的数据
        return {
          url: window.location.href,
          title: document.title,
          // ... 更多数据
        };
      }
    });
    
    return results[0].result;
  }
}
```

##### 批量采集任务管理器
**文件位置**：`src/entrypoints/background/batchCollectManager.ts`

**功能职责**：
- 管理批量采集任务队列
- 控制采集节奏（随机间隔）
- 处理暂停/继续/取消
- 进度通知

**核心代码**：
```typescript
export class BatchCollectManager {
  private taskQueue: CollectTask[] = [];
  private isRunning: boolean = false;
  private isPaused: boolean = false;
  private currentIndex: number = 0;
  private tabManager: TabManager;
  
  constructor() {
    this.tabManager = new TabManager();
  }
  
  async startBatchCollect(urls: string[]): Promise<void> {
    // 解析URL并创建任务
    this.taskQueue = urls
      .map(url => parseUrl(url))
      .filter(Boolean)
      .map(parsed => ({
        url: parsed!.originalUrl,
        parsed: parsed!,
        status: 'pending' as const,
        retryCount: 0
      }));
    
    this.isRunning = true;
    this.currentIndex = 0;
    
    await this.processQueue();
  }
  
  private async processQueue(): Promise<void> {
    while (this.isRunning && this.currentIndex < this.taskQueue.length) {
      // 检查是否暂停
      if (this.isPaused) {
        await sleep(1000);
        continue;
      }
      
      const task = this.taskQueue[this.currentIndex];
      
      try {
        // 采集数据
        const result = await this.collectTask(task);
        
        // 更新进度
        this.notifyProgress({
          total: this.taskQueue.length,
          current: this.currentIndex + 1,
          success: result.success ? 1 : 0,
          failed: result.success ? 0 : 1,
          currentUrl: task.url,
          status: 'running'
        });
        
        // 随机间隔（最后一个不需要等待）
        if (this.currentIndex < this.taskQueue.length - 1) {
          const interval = this.getRandomInterval();
          await sleep(interval);
        }
      } catch (error) {
        console.error('采集失败:', task.url, error);
      }
      
      this.currentIndex++;
    }
    
    // 采集完成
    this.isRunning = false;
    this.notifyComplete();
  }
  
  private async collectTask(task: CollectTask): Promise<CollectResult> {
    let tabId: number | null = null;
    
    try {
      // 创建后台标签页
      tabId = await this.tabManager.createBackgroundTab(task.url);
      
      // 等待页面加载
      await this.tabManager.waitForTabComplete(tabId);
      
      // 提取数据
      const data = await this.tabManager.extractDataFromTab(tabId);
      
      // 保存数据
      await this.saveData(task.parsed, data);
      
      return { success: true, url: task.url, data };
    } catch (error) {
      return { 
        success: false, 
        url: task.url, 
        error: error instanceof Error ? error.message : '未知错误' 
      };
    } finally {
      // 关闭标签页
      if (tabId) {
        await this.tabManager.closeTab(tabId);
      }
    }
  }
  
  private getRandomInterval(): number {
    const min = 3000;  // 3秒
    const max = 10000; // 10秒
    return Math.floor(Math.random() * (max - min + 1)) + min;
  }
  
  pause(): void {
    this.isPaused = true;
  }
  
  resume(): void {
    this.isPaused = false;
  }
  
  cancel(): void {
    this.isRunning = false;
    this.isPaused = false;
    this.taskQueue = [];
  }
  
  private notifyProgress(progress: BatchCollectProgress): void {
    // 通知所有监听者进度更新
    chrome.runtime.sendMessage({
      type: 'batch:collect:progress',
      data: progress
    });
  }
  
  private notifyComplete(): void {
    chrome.runtime.sendMessage({
      type: 'batch:collect:complete',
      data: {
        total: this.taskQueue.length,
        success: this.taskQueue.filter(t => t.status === 'completed').length,
        failed: this.taskQueue.filter(t => t.status === 'failed').length
      }
    });
  }
}
```

```typescript
      <img src="${post.coverUrl || ''}" class="cover" />
      <div class="info">
        <div class="title">${post.title || '无标题'}</div>
        <div class="meta">
          <span>@${post.authorName || '未知'}</span>
          <span>❤️ ${post.likeCount || 0}</span>
        </div>
      </div>
    `;
    
    const checkbox = item.querySelector('.checkbox') as HTMLInputElement;
    checkbox?.addEventListener('change', (e) => {
      const target = e.target as HTMLInputElement;
      if (target.checked) {
        this.selectedIds.add(id);
      } else {
        this.selectedIds.delete(id);
      }
      this.updateSelectedCount();
    });
    
    return item;
  }
  
  private selectAll(): void {
    this.cachedPosts.forEach((_, id) => {
      this.selectedIds.add(id);
    });
    this.updateCheckboxes();
    this.updateSelectedCount();
  }
  
  private deselectAll(): void {
    this.selectedIds.clear();
    this.updateCheckboxes();
    this.updateSelectedCount();
  }
  
  private updateCheckboxes(): void {
    const checkboxes = this.overlay.shadowRoot?.querySelectorAll('.checkbox');
    checkboxes?.forEach((checkbox) => {
      const item = (checkbox as HTMLElement).closest('.post-item');
      const id = item?.dataset.id;
      (checkbox as HTMLInputElement).checked = this.selectedIds.has(id || '');
    });
  }
  
  private updateSelectedCount(): void {
    const countEl = this.overlay.shadowRoot?.querySelector('#selectedCount');
    if (countEl) {
      countEl.textContent = `已选择 ${this.selectedIds.size} 条`;
    }
  }
  
  private async startBatchCollect(): Promise<void> {
    if (this.selectedIds.size === 0) {
      alert('请先选择要采集的帖子');
      return;
    }
    
    const posts = Array.from(this.selectedIds)
      .map(id => this.cachedPosts.get(id))
      .filter(Boolean) as PostEntity[];
    
    const progressEl = this.overlay.shadowRoot?.querySelector('#progress');
    const progressBar = progressEl?.querySelector('.progress-bar') as HTMLElement;
    const progressText = progressEl?.querySelector('.progress-text');
    
    if (progressEl) {
      progressEl.style.display = 'block';
    }
    
    let successCount = 0;
    let failCount = 0;
    
    for (let i = 0; i < posts.length; i++) {
      const post = posts[i];
      
      try {
        const response = await sendMessage('collect:post', {
          platform: 'xhs',
          post
        });
        
        if (response.success) {
          successCount++;
        } else {
          failCount++;
        }
      } catch (error) {
        failCount++;
      }
      
      // 更新进度
      const progress = ((i + 1) / posts.length) * 100;
      if (progressBar) {
        progressBar.style.width = `${progress}%`;
      }
      if (progressText) {
        progressText.textContent = `采集中... ${i + 1}/${posts.length}`;
      }
    }
    
    // 显示结果
    alert(`采集完成！成功: ${successCount}, 失败: ${failCount}`);
    this.close();
  }
}
```

## 八、总结

本文档详细梳理了批量采集功能的两种实现方向及其技术方案。核心要点：

### 8.1 两种批量采集方向对比

| 对比维度 | 方向一：页面缓存批量采集 | 方向二：URL批量采集 |
|---------|----------------------|------------------|
| **实现方式** | 从当前页面已缓存的数据中选择采集 | 批量输入URL，系统自动访问采集 |
| **适用场景** | 用户在浏览列表页时快速采集 | 用户有明确的采集目标列表 |
| **数据来源** | XHR拦截缓存的页面数据 | 后台标签页实时访问 |
| **采集速度** | 快（数据已缓存） | 较慢（需访问页面，3-10秒间隔） |
| **精准度** | 采集当前看到的内容 | 精准采集指定URL |
| **资源消耗** | 低 | 中等（需打开标签页） |
| **开发复杂度** | 中等 | 较高 |
| **用户体验** | 快速便捷 | 灵活精准 |
| **推荐优先级** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

### 8.2 技术方案总结

**方向一核心技术**：
- 浮动按钮 + 批量采集面板
- Shadow DOM样式隔离
- 批量选择和进度反馈
- 消息通信机制

**方向二核心技术**：
- URL解析和验证
- 后台标签页管理
- 随机间隔机制（3-10秒）
- 任务队列管理
- 进度实时通知

### 8.3 实施建议

**推荐实施顺序**：
1. **优先实现方向一**（页面缓存批量采集）
   - 开发周期短（约6天）
   - 用户需求更迫切
   - 技术复杂度较低
   
2. **其次实现方向二**（URL批量采集）
   - 开发周期约9天
   - 满足精准采集需求
   - 技术挑战较大

**关键成功因素**：
- ✅ 清晰的模块划分和职责定义
- ✅ 完善的错误处理和重试机制
- ✅ 友好的用户交互和进度反馈
- ✅ 合理的随机间隔避免被封
- ✅ 充分的测试和性能优化
