# 智云台 - 小红书蒲公英&星图AI助手 项目梳理

## 一、项目概述

**项目名称**：智云台 - 小红书蒲公英&星图AI助手\
**项目类型**：Chrome浏览器扩展插件（Manifest V3）\
**当前版本**：4.8.2\
**核心定位**：专为媒介与PR人员打造的小红书&蒲公英、抖音&星图达人选号工具

**核心价值**：

- 让达人筛选变得高效、智能、不再繁琐
- 支持多平台数据采集与分析
- AI辅助审号，提升工作效率

**支持平台**：

- 小红书（[www.xiaohongshu.com）](http://www.xiaohongshu.com）)
- 蒲公英平台（pgy.xiaohongshu.com）
- 抖音（[www.douyin.com）](http://www.douyin.com）)
- 星图平台（[www.xingtu.cn）](http://www.xingtu.cn）)
- 自有平台（ts-martech.com）

## 二、技术架构

### 2.1 技术栈

| 技术层级 | 技术选型                         | 说明          |
| ---- | ---------------------------- | ----------- |
| 扩展框架 | Chrome Extension Manifest V3 | 最新扩展标准      |
| 前端框架 | React 18.3.1                 | 侧边栏UI开发     |
| UI架构 | Side Panel API               | Chrome原生侧边栏 |
| 脚本注入 | Content Scripts              | 页面数据采集      |
| 后台服务 | Service Worker               | 消息中转与处理     |
| 网络请求 | declarativeNetRequest        | 声明式网络请求拦截   |

### 2.2 架构设计

```
┌─────────────────────────────────────────┐
│      Background Service Worker          │
│  (消息路由、存储管理、网络拦截)           │
└──────────────┬──────────────────────────┘
               │ Chrome Messaging API
    ┌──────────┼──────────┐
    │          │          │
┌───▼───┐  ┌───▼───┐  ┌───▼───┐
│ Side  │  │Content│  │ Main  │
│ Panel │  │Script │  │ World │
│  UI   │  │       │  │Script │
└───────┘  └───────┘  └───────┘
```

**核心设计模式**：

1. **消息驱动架构**：通过Chrome Messaging API实现各组件间通信
2. **注入模式**：Content Script注入到目标页面，MAIN world脚本获取页面上下文
3. **代理模式**：Background代理跨域请求，解决Content Script跨域限制
4. **观察者模式**：监听XHR请求、页面变化，实时采集数据

## 四、核心功能模块

### 4.1 平台识别与路由

**功能描述**：自动识别当前页面所属平台，启用对应功能模块

**支持平台**：

- **抖音平台**：返回平台ID 0
  - [www.douyin.com](http://www.douyin.com)
  - [www.xingtu.cn](http://www.xingtu.cn)
  - yestar.ts-martech.com
- **小红书平台**：返回平台ID 1
  - pgy.xiaohongshu.com
  - [www.xiaohongshu.com](http://www.xiaohongshu.com)
  - xhs.ts-martech.com
- **自有平台**：返回平台ID 2
  - \*.ts-martech.com

**核心代码**：

```javascript
const getPlatformId = (url = location.href) => {
  if (url.startsWith("https://www.xingtu.cn") || 
      url.startsWith("https://www.douyin.com") ||
      url.startsWith("https://yestar.ts-martech.com")) return 0;
  
  if (url.startsWith("https://pgy.xiaohongshu.com") || 
      url.startsWith("https://www.xiaohongshu.com") ||
      url.startsWith("https://xhs.ts-martech.com")) return 1;
  
  if (isTsMartech(url)) return 2;
};
```

### 4.2 数据采集系统

#### 4.2.1 XHR请求拦截

**实现原理**：

- 重写 `XMLHttpRequest.prototype.open` 和 `XMLHttpRequest.prototype.send`
- 拦截所有XHR请求的响应数据
- 过滤JSON格式响应，提取关键数据

**采集数据类型**：

- 小红书笔记详情
- 小红书用户主页数据
- 抖音视频数据
- 评论数据
- 达人信息

**核心代码**：

```javascript
XMLHttpRequest.prototype.open = function() {
  if (arguments && arguments.length > 0) {
    this._method = arguments[0];
  }
  originalOpen.apply(this, arguments);
};

XMLHttpRequest.prototype.send = function() {
  const originalOnload = this.onload;
  this.onload = function() {
    interceptResponse(this);
    originalOnload && originalOnload.apply(this, arguments);
  };
  originalSend.apply(this, arguments);
};
```

#### 4.2.2 页面初始数据采集

**采集时机**：页面加载完成时

**采集内容**：

- 小红书探索页笔记详情（`window.__INITIAL_STATE__.note`）
- 小红书用户主页数据（`window.__INITIAL_STATE__.user`）

### 4.3 侧边栏UI系统

**UI架构**：基于React 18开发的侧边栏界面

**主要功能**：

- 达人筛选与展示

**交互方式**：

- 快捷键 `Ctrl+I` 激活扩展

### 4.4 Background消息中转

**核心职责**：

1. **消息路由**：处理来自Content Script和Side Panel的消息
2. **跨域请求代理**：代理Content Script的跨域fetch请求
3. **存储管理**：管理Chrome Storage数据
4. **标签页通信**：向指定标签页发送消息

**消息协议**：

```typescript
interface MessageProtocol {
  paths: string[];    // 调用路径，如 ["fetch"]
  args: any[];        // 调用参数
}
```

**核心功能**：

- `fetch`：跨域HTTP请求代理
- `storage-changed`：存储变化通知
- `sidebar-open`：打开侧边栏
- `commands-command`：快捷键命令处理

### 4.5 页面截图与渲染

**功能描述**：提供页面元素截图能力

**核心技术**：

- html2canvas：将DOM元素转换为Canvas
- Canvas API：图像处理与导出
- 外部资源嵌入：字体、图片等

**应用场景**：

- 达人主页截图
- 数据报告生成
- 竞品分析报告

## 五、核心能力详解

### 5.1 网络请求拦截

**实现方式**：declarativeNetRequest API

**配置文件**：douyin.json

**拦截规则**：

```json
{
  "id": 1,
  "priority": 1,
  "action": {
    "type": "modifyHeaders",
    "responseHeaders": [
      {
        "header": "Content-Security-Policy",
        "operation": "remove"
      },
      {
        "header": "X-Frame-Options",
        "operation": "remove"
      }
    ]
  },
  "condition": {
    "urlFilter": "||douyin.com/user/*",
    "resourceTypes": ["sub_frame"]
  }
}
```

**作用**：

- 移除CSP限制，允许内嵌iframe
- 移除X-Frame-Options，支持跨域iframe嵌入

### 5.2 数据导出能力

**导出类型**：

- 笔记/视频数据
- 评论数据
- 达人数据

**导出格式**：Excel（通过后端API生成）

**导出流程**：

1. Content Script采集数据
2. 发送到Background
3. Background调用后端API
4. 生成Excel文件
5. 触发浏览器下载

### 5.3 AI审号功能

**功能描述**：利用AI技术自动审核达人账号

**审核维度**：

- 账号质量评估
- 内容质量分析
- 粉丝画像分析
- 商业价值评估

**实现方式**：调用后端AI服务接口

### 5.4 竞品分析

**功能描述**：分析竞品账号数据，生成对比报告

**分析维度**：

- 粉丝增长趋势
- 内容发布频率
- 互动数据对比
- 商业合作情况

### 5.5 批量邀约

**功能描述**：批量向达人发送合作邀约

**实现方式**：

- 集成蒲公英/星图平台API
- 自动填写邀约表单
- 批量提交邀约请求

## 八、数据流程

### 8.1 数据采集流程

```
用户访问平台页面
    ↓
Content Script注入
    ↓
拦截XHR请求 / 读取页面数据
    ↓
数据预处理与过滤
    ↓
发送到Background
    ↓
Background转发到后端API
    ↓
数据持久化存储
```

### 8.2 用户交互流程

```
用户点击扩展图标
    ↓
Background判断当前平台
    ↓
发送消息到Content Script
    ↓
Content Script打开侧边栏
    ↓
用户在侧边栏操作
    ↓
Side Panel发送请求到Background
    ↓
Background调用对应API
    ↓
返回结果并更新UI
```

## 九、技术亮点

### 9.1 多平台适配

- **统一抽象**：通过平台ID实现多平台统一管理
- **模块化设计**：各平台功能独立，易于扩展
- **配置驱动**：通过配置文件管理平台差异

### 9.2 数据采集优化

- **无感采集**：通过拦截XHR请求，无需额外请求
- **实时更新**：监听页面变化，实时采集最新数据
- **缓存机制**：避免重复采集，提升性能

### 9.3 用户体验优化

- **侧边栏设计**：不遮挡页面内容，体验流畅
- **快捷键支持**：提升操作效率
- **自动识别**：无需手动切换平台

### 9.4 安全性设计

- **权限最小化**：仅申请必要权限
- **数据加密**：敏感数据加密传输
- **域名白名单**：限制脚本注入范围
