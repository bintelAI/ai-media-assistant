## 一、产品概述

### 1.1 产品定位

智联采集是一款面向媒介、PR、运营人员的专业社媒数据采集与分析工具，通过浏览器扩展插件形式，实现小红书、抖音、快手等多平台的数据采集、AI分析、批量管理等功能。

### 1.2 核心价值主张

| 价值维度 | 描述 |
|---------|------|
| **高效采集** | 采用先进的脚本注入技术，稳定采集多平台数据 |
| **智能分析** | AI辅助审号、竞品分析、数据洞察 |
| **专业工具** | 批量邀约、数据导出、团队协作 |
| **离线可用** | 核心功能离线可用，数据本地处理 |
| **用户友好** | 现代化UI设计，流畅的交互体验 |

### 1.3 目标用户

| 用户群体 | 核心需求 | 使用场景 |
|---------|---------|---------|
| 媒介专员 | 达人筛选、批量邀约、数据整理 | 日常达人合作管理 |
| PR人员 | 舆情监控、竞品分析、效果评估 | 品牌传播管理 |
| 运营人员 | 内容分析、数据监控、效果追踪 | 日常运营决策 |
| 数据分析师 | 数据采集、报表生成、趋势分析 | 数据驱动决策 |

### 1.4 支持平台矩阵

| 平台 | 域名 | 采集能力 | AI分析 | 批量操作 |
|------|------|---------|--------|---------|
| 小红书 | www.xiaohongshu.com | ✅ 完整支持 | ✅ 支持 | ✅ 支持 |
| 蒲公英 | pgy.xiaohongshu.com | ✅ 完整支持 | ✅ 支持 | ✅ 支持 |

### 2.3 媒体下载功能

#### 2.3.1 无水印下载

**功能描述**：下载无水印的视频和图片

**支持格式**：

| 媒体类型 | 支持格式 | 下载方式 |
|---------|---------|---------|
| 视频 | MP4 | 浏览器下载 |
| 图片 | JPG/PNG | 浏览器下载 |
| 图文集 | ZIP打包 | 批量打包下载 |

**文件命名规则**：

| 命名变量 | 变量说明 | 示例 |
|---------|---------|------|
| {platform} | 平台名称 | xiaohongshu |
| {author} | 作者昵称 | 张三 |
| {title} | 作品标题 | 我的第一篇笔记 |
| {date} | 发布日期 | 20260327 |
| {id} | 作品ID | abc123 |

**命名模板示例**：
- `{platform}_{author}_{date}_{title}.mp4`
- `{platform}_{author}_{id}.jpg`

#### 2.3.2 批量下载

**功能描述**：批量下载多个作品媒体文件

**批量策略**：

| 参数名称 | 参数说明 | 取值范围 |
|---------|---------|---------|
| 并发数 | 同时下载数量 | 1-5 |

| 间隔时间 | 下载间隔（秒） | 0-10 |

**下载队列管理**：

```
添加下载任务
    ↓

    ↓
并发控制调度
    ↓
执行下载任务
    ↓
成功 → 完成

    ↓
重试次数用尽 → 标记失败

## 三、技术架构

| 原则名称 | 原则说明 |
|---------|---------|

| 策略模式 | 不同平台采用策略模式适配 |
| 双层架构 | ISOLATED + MAIN World双层架构 |
| 代理模式 | Background代理跨域请求 |
| 观察者模式 | 监听页面变化和请求响应 |

```

│  - 跨域请求代理                                       │
│  - 数据缓存管理                                       │
└────────────────────┬────────────────────────────────┘

      ┌──────────────┼──────────────┐
      │              │              │
┌─────▼─────┐  ┌─────▼─────┐  ┌─────▼─────┐
│ Side Panel│  │ Content   │  │   MAIN    │

│           │  │ React应用  │  │ 平台API   │

#### 3.3.1 前端框架选型分析

**对比方案**：

| 对比维度 | React 18 | Vue 3 | Svelte |
|---------|:--------:|:-----:|:------:|
| 生态成熟度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| TypeScript支持 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 组件库丰富度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| Chrome扩展适配 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 参考项目使用 | ✅ 两个都用 | ❌ | ❌ |

**选型结论**：React 18

**选型理由**：

| 理由 | 说明 |
|------|------|
| 参考项目一致 | 社媒助手和靓号都使用React，可直接借鉴代码 |
| 生态最成熟 | 组件库、工具链、社区支持最完善 |
| TypeScript友好 | 类型支持最完善，适合企业级项目 |
| Chrome扩展适配 | WXT框架对React支持最好 |
| 团队熟悉度 | 当前项目已使用React，无迁移成本 |

#### 3.3.2 UI组件库选型分析

**对比方案**：

| 组件库 | 技术栈 | 包体积 | 设计风格 | 可定制性 | 推荐指数 |
|-------|--------|--------|---------|---------|---------|
| Radix UI | React | 小 | 无样式 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| Shadcn/ui | React | 小 | 现代简约 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| Ant Design | React | 大 | 企业风 | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| MUI | React | 中 | Material | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

**选型结论**：Radix UI + Tailwind CSS（或 Shadcn/ui）

**选型理由**：

| 理由 | 说明 |

#### 3.3.5 Side Panel UI方案详解

**技术方案**：Chrome 原生 Side Panel API

**什么是 Side Panel API**：

Side Panel API 是 Chrome 在 Manifest V3 中引入的原生侧边栏功能，允许扩展在浏览器侧边栏中显示自定义UI。

**实现原理**：

```
用户点击扩展图标 / 快捷键
    ↓
Background Service Worker
    ↓
调用 chrome.sidePanel.open()
    ↓
Chrome 浏览器原生侧边栏
    ↓
加载 sidepanel.html（React应用）
```

**核心配置**：

| 配置项 | 配置内容 |
|-------|---------|
| 权限声明 | "sidePanel" |
| 入口配置 | side_panel.default_path |
| Chrome版本要求 | Chrome 114+ |

**与其他方案对比**：

| 对比维度 | Side Panel API | Content Script注入 | Popup弹窗 |
|---------|:--------------:|:------------------:|:---------:|
| 显示位置 | 浏览器侧边栏 | 页面内部 | 点击弹出 |
| 遮挡页面 | ❌ 不遮挡 | ✅ 可能遮挡 | ✅ 遮挡 |
| 切换标签 | 保持显示 | 需重新注入 | 自动关闭 |
| 空间大小 | 充足 | 受限 | 很小 |
| 用户体验 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| 开发复杂度 | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ |

**核心优势**：

| 优势 | 说明 |
|------|------|
| 原生体验 | 浏览器原生支持，体验流畅 |
| 不遮挡页面 | 侧边栏独立于网页，不影响浏览 |
| 持久显示 | 切换标签页时保持显示 |
| 响应式 | 自动适应窗口大小 |
| 安全性高 | 独立的扩展上下文 |

**选型理由**：

| 理由 | 说明 |
|------|------|
| 用户体验最佳 | 不遮挡页面，独立显示 |
| 原生支持 | Chrome官方推荐方案 |
| 开发简单 | 无需处理复杂的DOM注入 |
| 性能优秀 | 浏览器原生渲染 |
| 未来趋势 | Manifest V3的标准方案 |
| 参考项目一致 | 社媒助手和靓号都使用此方案 |

### 3.4 核心技术方案

#### 3.4.1 脚本注入技术

**技术方案**：采用 declarative_net_request API 重定向平台脚本

**实现原理**：

```
平台加载原始脚本
    ↓

    ↓
重定向到扩展修改后的脚本
    ↓
注入自定义代码绑定平台函数
    ↓
暴露平台API供扩展调用
```

**注入策略**：

| 平台 | 拦截脚本 | 注入内容 |
|------|---------|---------|
| 抖音 | client-entry.js | 绑定HTTP函数 |
| 小红书 | vendor-dynamic.js | 获取registerHttp函数 |
| 快手 | 主入口脚本 | 绑定签名函数 |

#### 3.4.2 页面UI注入技术

**技术方案**：Content Script 动态注入页面UI元素

**什么是页面UI注入**：

通过 Content Script 在目标网页上动态插入自定义UI元素（按钮、悬浮框、工具栏等），实现与用户的无缝交互。

**实现原理**：

```
用户访问平台页面
    ↓
Content Script自动注入
    ↓
监听页面加载完成
    ↓
识别页面类型（作品页/用户主页/列表页）
    ↓
根据页面类型选择注入策略
    ↓
在指定位置创建DOM元素
    ↓
使用Shadow DOM隔离样式
    ↓
绑定交互事件
    ↓
触发对应功能
```

**注入位置策略**：

| 页面类型 | 注入位置 | 注入元素 | 触发功能 |
|---------|---------|---------|---------|
| 作品详情页 | 作品标题旁/操作栏 | 采集按钮、下载按钮 | 一键采集、无水印下载 |
| 用户主页 | 关注按钮旁/简介区域 | 采集按钮、AI审号按钮 | 采集创作者、AI分析 |
| 作品列表页 | 作品卡片悬浮层 | 批量采集按钮 | 批量采集作品 |
| 评论区域 | 评论列表顶部 | 采集评论按钮 | 批量采集评论 |
| 蒲公英/星图 | 达人卡片上 | 快捷操作按钮 | 快速邀约、收藏 |

**核心技术要点**：

| 技术要点 | 说明 |
|---------|------|
| Shadow DOM | 隔离注入元素的样式，避免与页面样式冲突 |
| MutationObserver | 监听页面DOM变化，动态注入UI |
| CSS隔离 | 使用scoped样式，避免污染页面 |
| 事件委托 | 高效处理动态元素的交互事件 |
| 位置计算 | 智能计算注入位置，避免遮挡 |

**与Side Panel的协同**：

```
页面注入按钮（快捷操作）
    ↓
点击触发采集
    ↓
数据传递到Side Panel
    ↓
Side Panel展示详细数据
    ↓
用户进行后续操作
```

**核心优势**：

| 优势 | 说明 |
|------|------|
| 操作便捷 | 无需打开侧边栏即可快速操作 |
| 上下文相关 | 根据当前页面智能显示相关功能 |
| 用户友好 | 减少操作步骤，提升效率 |
| 视觉统一 | 与平台UI风格融合 |

**与社媒助手的区别**：

| 对比维度 | 靓号（有页面注入） | 社媒助手（无页面注入） |
|---------|:-----------------:|:--------------------:|
| 操作入口 | 页面按钮 + Side Panel | 仅Side Panel |
| 操作便捷性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 用户感知度 | 高，直观可见 | 中，需主动打开 |
| 开发复杂度 | ⭐⭐⭐⭐ | ⭐⭐⭐ |

**推荐采用**：✅ 采用页面UI注入技术，提升用户体验
