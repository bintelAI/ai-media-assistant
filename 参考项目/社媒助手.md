**技术要点**：

#### 小红书平台 - vendor-dynamic.js
```javascript
// 注入 registerHttp 函数
function injectRegisterHttpSimple(a) {
  const re = /function\s+registerHttp\s*\(\s*([A-Za-z_$][\w$]*)\s*\)\s*\{\s*/;
  return a.replace(re, (match, param) => {
    const injection = `window._smzsRequest=${param};`;
    return match + injection;
  });
}
```

**技术要点**：
- 拦截小红书的 vendor-dynamic.js 脚本
- 注入代码获取平台的请求函数引用
- 通过 `window._smzsRequest` 暴露给扩展使用

### 4.2 请求代理机制

#### Background Service Worker
```javascript
// 消息处理定义
defineMessage("fetch", async ({ data }) => {
  // 跨域请求代理
  return fetch(data.url, data);
});

defineMessage("executeScript", async ({ data, sender }) => {
  // 在 MAIN world 执行脚本
  const result = await chrome.scripting.executeScript({
    target: { tabId: sender.tab.id },
    world: "MAIN",
    func: (params, code) => {
      const AsyncFunction = Object.getPrototypeOf(async function(){}).constructor;
      return new AsyncFunction(...params)(...args);
    },
    args: [[...params, code], data.args || []]
  });
});
```

**核心能力**：
- **跨域请求代理**：Content Script无法直接跨域，通过Background代理
- **脚本动态执行**：在页面上下文中动态执行JavaScript代码
- **Chrome API封装**：统一封装Chrome扩展API调用

// XOR加密类
class Encryptor {
  constructor() {
    this.key = this.generateRandomKey(16);
  }
  
  encrypt(data) {
    const bytes = new TextEncoder().encode(data);
    const encrypted = new Uint8Array(bytes.length);
    for (let i = 0; i < bytes.length; i++) {

## 六、数据流程

### 6.1 数据采集流程

```
用户操作
    ↓
Content Script (ISOLATED)
    ↓ 发送消息
Background Service Worker
    ↓ 执行脚本
MAIN World Script
    ↓ 调用平台API
平台服务器
    ↓ 返回数据
MAIN World Script (拦截响应)
    ↓ 转发数据
Content Script (处理数据)
    ↓ 导出/下载
本地文件/飞书
```

### 6.2 请求拦截流程

```
平台页面发起请求
    ↓
拦截 fetch/XMLHttpRequest
    ↓
捕获请求参数和响应数据
    ↓
通过 postMessage 发送给 Content Script
    ↓
Content Script 处理数据
```

## 十一、总结

社媒助手 v3.1.3 是一个技术成熟的商业级浏览器扩展，其核心优势在于：

1. **创新的脚本注入技术**：通过重定向平台脚本实现深度集成
2. **稳定的API调用方式**：直接使用平台原生函数，避免逆向工程
3. **完善的架构设计**：双层脚本架构兼顾安全性和功能性
4. **丰富的功能支持**：数据采集、媒体下载、数据导出一体化

这些技术方案对于开发类似的数据采集工具具有重要的参考价值。
