# 架构设计文档编写规范

本规范旨在标准化项目中的所有“架构设计图”及“需求设计文档”，确保其结构清晰、逻辑严密，并具备高度的可读性和可维护性。AI 在生成此类文档时，必须严格参照此模板和逻辑进行梳理。

## 1. 核心设计原则
- **明确统一立场**：文档开头必须声明当前的全局逻辑立场（例如：默认隐式授权、默认拒绝、默认放行等）。
- **隐式与显式结合**：清晰界定“隐式逻辑”（如角色默认能力）与“显式配置”（如数据库特定记录）的生效边界。
- **层级化判定**：展示逻辑判定时的分层顺序（如：角色层 -> 表级层 -> 行级层 -> 列级层）。

## 2. 文档必备章节结构

### 2.1 标题与背景说明
- **标题**：清晰定义设计的核心功能（如：权限机制架构设计图）。
- **背景说明**：简述文档解决的核心问题、业务场景及当前系统采用的统一口径。

### 2.2 系统架构图 (Mermaid - flowchart LR)
- **要求**：使用 `mermaid` 的 `flowchart LR` 绘制。
- **内容**：涵盖用户、前端层（React/Hooks/Store）、后端层（Controller/Service/Middleware）、数据库层（Tables），并用连线表示核心数据流向。

### 2.3 数据库设计 (Mermaid - erDiagram)
- **ER 图**：展示核心表的关联关系（1:1, 1:N, M:N）。
- **数据字典**：以表格形式详述关键字段的：值、中文名称、说明、适用场景/枚举定义。

### 2.4 判定模型/流程图 (Mermaid - flowchart TD)
- **要求**：使用 `mermaid` 的 `flowchart TD` 绘制。
- **逻辑**：清晰展示“开始 -> 条件分支 -> 最终结果”的完整判定链条。特别强调“无数据时”与“有数据时”的不同分支。

### 2.5 交互时序图 (Mermaid - sequenceDiagram)
- **内容**：展示用户发起请求后，前端、后端、Service、数据库之间的调用顺序及返回结果。

### 2.6 问题定位图/排查口径 (Mermaid - flowchart TD)
- **目的**：针对常见业务痛点（如“明明有权限但只读”）提供结构化的排查逻辑。
- **逻辑**：从最基础的条件开始，逐级下钻到细节原因。

### 2.7 业务流程/配置路径 (Mermaid - flowchart LR)
- **内容**：展示从管理员配置到用户最终看到结果的完整链路。

### 2.8 统一口径/规则定义
- **按能力分项**：对每个核心模块（如：CRUD、字段控制、菜单获取）给出明确的“统一定义”和“排查口径”。
- **表格对比**：通过“权限层 | 数据表 | 主要职责 | 无数据时 | 有数据时”的对照表，消除模糊性。

### 2.9 项目通用封装函数及功能说明
- **背景**：为了提高开发效率和代码统一性，本项目在 `src/comm` 目录及各模块中提供了一系列封装好的公共函数和中间件。
- **核心工具类 (Utils)**：
  - **调用方式**：通过 `@Inject() utils: Utils` 注入后使用。
  - **关键方法**：
    - `md5(str)`: MD5 加密。
    - `matchUrl(pattern, url)`: 支持 `*` 和 `**` 的 URL 匹配。
    - `getUUID()`: 生成 UUID。
    - `sleep(ms)`: 异步延迟。
    - `getReqIP(ctx)`: 获取客户端真实 IP。
    - `toCamelCase(obj)`: 下划线转小驼峰。
    - `extractJSONFromText(text)`: 自动从混合文本中提取并解析 JSON。
- **路径与系统工具**：
  - `pDataPath()` / `pUploadPath()`: 获取系统数据及上传文件的物理路径。
  - `availablePort(startPort)`: 探测可用 TCP 端口。
- **业务中间件**：
  - `BaseAuthorityMiddleware`: 负责后台 `/admin/` 接口的 JWT 及权限校验。
  - `BaseLogMiddleware`: 自动记录接口操作日志。
  - `BaseTranslateMiddleware`: 响应数据动态翻译（国际化）。

## 3. 视觉与排版要求
- **Mermaid 规范**：所有图表必须包含在 ` ```mermaid ` 代码块中，且确保语法正确。
- **表格清晰**：使用标准 Markdown 表格展示字段、数据字典及规则对比。
- **重点标注**：使用 `**粗体**` 或 `***` 分隔符突出核心口径和注意点。
